Software Development Life Cycle (SDLC):

Explain the various phases of the Software Development Life Cycle. Provide a brief description of each phase.
1. Requirement- it involves gathering and documenting user needs and system requirements.
2. Design- Here there is creating of high-level and detailed designs of the software architecture and user interface
3. Implementation-Involves writing a code and building software according to the design specifications.
4. Testing-Conducting various tests to ensure the software meets quality standards and functional requirements
5. Deployment- Releasing the software to users or the customers
6. Maintainance- Provide ongoing updates and patches after deployment.  
Agile vs. Waterfall Models:

Compare and contrast the Agile and Waterfall models of software development. What are the key differences, and in what scenarios might each be preferred?

Agile model is an iterative and incremental approach to project management and software development. It emphasizes felexibility, collaboration and customer feedback. This model is convinient for long term projects.
Waterfall model is a linear and sequential model which means that each phase must be completed before moving to the next phase. This model is recommended for short term projects. 


Requirements Engineering:

What is requirements engineering? Describe the process and its importance in the software development lifecycle.

This is an important phase in SDLC that involves the systematic process of identifying, documenting, analyzing, and managing the needs and requirements of stakeholders for a software system.
Process
Elicitation- To gather requirements from stakeholders, including customers, end-users, and other relevant parties.
Analysis- To understand and clarify the gathered requirements, resolving any ambiguities or conflicts.
Specification- To formally document the requirements in a clear and detailed manner.
Validation- To ensure that the documented requirements accurately reflect the stakeholders' needs and are achievable within the project's constraints.
Management- To handle changes to requirements throughout the project lifecycle.
Requirements engineering provides the basis for designing, developing, and testing software. Clear and well-documented requirements ensure that developers understand what needs to be built.

Software Design Principles:

Explain the concept of modularity in software design. How does it improve maintainability and scalability of software systems?
Modularity in software design is a principle that involves dividing a software system into independent modules, each responsible for a specific part of the system's functionality. These modules interact through well-defined interfaces but are otherwise self-contained. The key concept is to encapsulate functionality within modules so that changes to one module have minimal impact on others.


Testing in Software Engineering:

Describe the different levels of software testing (unit testing, integration testing, system testing, acceptance testing). Why is testing crucial in software development?
Unit testing-Unit testing involves testing individual components or functions of a software application in isolation to ensure that each part works correctly.
Integration testing-Integration testing checks the interactions between different modules or components of a software application.
System testing-System testing evaluates the complete and integrated software system to verify that it meets the specified requirements.
Testing is crucial because developers can systematically identify and address issues, validate requiremnents and ensure that the final product meets user expectations and business needs. 

Version Control Systems:

What are version control systems, and why are they important in software development? Give examples of popular version control systems and their features.
Version control systems (VCS) are tools that help manage changes to source code and other collections of files over time. They enable multiple developers to work on a project simultaneously, keep track of every modification to the code, and facilitate collaboration and coordination among team members.
Examples
Git- Distributed VCS which means that each developer has a copy of the repository including its history.
     Lightweight and fast performance.
Subversion-Centralized VCS, a single central repository. 
           Support atomic commits meaning all changes in a commit are applied together. 
Mercurial- Distributed VCS like git but designed to be easier to learn and use
           Strong branching and merging support.                 

Software Project Management:

Discuss the role of a software project manager. What are some key responsibilities and challenges faced in managing software projects?
The role of a software project manager is overseeing and guiding software development projects from inception to completion. They ensure that projects are completed within stipulated time and budget and meets the specified requirements and quality standards. 
Key responsibilities include:
Planning and scheduling- this process defines the projects scope, objectives and deliverables. 
Resource management- Identify and allocate resources required for the project. 
Risk management- identify potential risks and develop mitigation strategies.
Challenges include:
Time management- balancing project deadlines with the reality of development delays and unexpected issues.
Resource contraints- dealing with limited resources and ensuring their efficient utilization
Technology changes-Keeping up with the rapid technological advancement and intergrating new ones as needed.

Software Maintenance:

Define software maintenance and explain the different types of maintenance activities. Why is maintenance an essential part of the software lifecycle?
Software maintenance is the process of modifying and updating software applications after their initial deployment.This process ensures that the software continues to function properly, remains secure and evolves to meet changing user needs and technological advancement. 
Types:
Corrective maintainance- Involves fixing bugs discovered in the software after it has been released.Which is important to correct errors that affect functionality, performance, or security.
Adaptive Maintenance-Entails modifying the software to work in new or changed environments,to ensure compatibility with evolving hardware, operating systems, software dependencies, or regulatory requirements.
Perfective Maintenance-Focuses on enhancing the software by improving performance, adding new features, or refining existing functionalities.
Preventive Maintenance-Involves making changes to the software to prevent future problems.

Maintainance is an important part in software lifescycle because it prolongs software longetivity which means that maintainance ensures that the software continues to funtion correctly and efficiently over time. 
It enhances performance by regular updates and optimizations which improves software perfomance making it faster and more efficient.
Maintenance allows the software to evolve in response to changing user needs, market conditions, and technological advancements.
Maintenance helps identify and fix security vulnerabilities, protecting the software from potential threats and ensuring compliance with security standards.

Ethical Issues:
Privacy violation- this involves developing softwares that invade users privacy or tracts their activities without transparency
Security risks- Neglecting to implement adequate security measures leading to vulnerabilities. 
Intellectual property- this is the use of copyrighting materials without consent which can involve plagiarizing of other developers codes or algorithm. 
Transaparency and accountability- Creating software with hidden functionalities by failing to disclose limitations or errors associated with the software.

How to adhere to ethical standards:
Following established codes of ethics, such as the ACM Code of Ethics or the IEEE Code of Ethics, which provide guidelines for professional conduct.
Be transparent about the capabilities, limitations, and risks of the software by communicating openly with the stakeholders users and colleagues about ethical concerns and decisions.
Implement robust data protection measures to safeguard user privacy for example obtaining informed concent from users before collecting, storing or using their data.
